# Rmarkdownでのログの例

目的、実行コマンドなどを丁寧に記載する。  
実験ノートのように、実施目的、実行内容、実行結果、自分で気づいたこと、ポイント、そこから自分でした解釈など、書けるだけ全て記載する。

コードチャンクの調整によく使うオプション

| オプション      | 意味                                     |
| --------------- | ---------------------------------------- |
| eval = FALSE    | 実行しない（コードは表示）               |
| echo = FALSE    | コードは隠して結果だけ表示               |
| include = FALSE | コードも結果も表示しない（内部実行のみ） |
| warning = FALSE | Warningを表示しない                      |
| message = FALSE | library()などのメッセージを表示しない    |

:warning: 今回はRではなくLinuxのコマンドなので、コードチャンクで実施するとエラーが出ます。  
:warning: `eval = FALSE`のオプションをつけることで、コマンドは実行せずにコードのみの記録を取ることができる
:warning: デフォルトではつけないほうがいい
